Properties screen

Description:

This is the Properties screen. On this screen the user can search for a property by providing the property’s address in the relevant search bar to view and maintain the property. The screen displays all the properties that exist on the system regardless of the property status.

Reference

Control

Functionality description

1

Screen header

Screen header titled “Properties”

2

Screen subheading

Screen subheading titled “view and maintain”

3

Add property button

This button opens the “Add Property” modal where the user can add a property.

4

Generate Property Report button

Opens the “Generate Property Report” modal where the user can generate a report based on data parameters

5

Search bar

Input of type Text to search through the grid list cards

6

Grid list card

A content container with a list of all the properties and a button to view and delete that specific property

7

Card title

Card Title with the property address

8

Card content

Card content with the suburb of the property

9

View property button

This button opens the “View Property” modal where the user can view the property details

10

Delete this property button

This button opens the “Delete this Property?” modal where the user can confirm to delete the property.
